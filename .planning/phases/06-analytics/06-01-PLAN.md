**Scan 2 — Avg by bean (RPT-03):**
```typescript
const beanScan = new GlideRecord('x_664529_aibrew_brew_log')
beanScan.addQuery('rating', '>=', '1')           // D-08: exclude unrated
beanScan.query()
const beanMap = new Map<string, { name: string; sum: number; count: number }>()
while (beanScan.next()) {
  const beanSysId = beanScan.getValue('bean') || ''
  if (!beanSysId) continue
  const beanName  = beanScan.getDisplayValue('bean') || beanSysId
  const rating    = parseInt(beanScan.getValue('rating') || '0', 10)
  if (!beanMap.has(beanSysId)) {
    beanMap.set(beanSysId, { name: beanName, sum: 0, count: 0 })
  }
  const entry = beanMap.get(beanSysId)!
  entry.sum   += rating
  entry.count += 1
}
const by_bean = Array.from(beanMap.values())
  .map(e => ({ name: e.name, avg: Math.round((e.sum / e.count) * 10) / 10, count: e.count }))
  .sort((a, b) => b.avg - a.avg)                // D-10: highest avg first
```

**Scan 3 — Avg by method (RPT-04):**
```typescript
const methodScan = new GlideRecord('x_664529_aibrew_brew_log')
methodScan.addQuery('rating', '>=', '1')         // D-08: exclude unrated
methodScan.query()
const methodMap = new Map<string, { sum: number; count: number }>()
while (methodScan.next()) {
  const method = methodScan.getValue('method') || ''
  if (!method) continue
  const rating = parseInt(methodScan.getValue('rating') || '0', 10)
  if (!methodMap.has(method)) {
    methodMap.set(method, { sum: 0, count: 0 })
  }
  const entry = methodMap.get(method)!
  entry.sum   += rating
  entry.count += 1
}
const by_method = Array.from(methodMap.entries())
  .map(([method, e]) => ({ method, avg: Math.round((e.sum / e.count) * 10) / 10, count: e.count }))
  .sort((a, b) => b.avg - a.avg)                // D-10: highest avg first
```

| T-06-01 | Elevation of Privilege | GET /api/x_664529_aibrew/analytics/summary | mitigate | `authentication: true` and `authorization: true` on the route (analytics-api.now.ts) — ServiceNow rejects requests without a valid session/g_ck before the handler script executes |
| T-06-02 | Information Disclosure | analytics-handler.ts GlideRecord queries | accept | No user-supplied input parameters in v1 (D-07) — handler builds queries with hardcoded field names and fixed thresholds (`rating >= 1`, `setLimit(30)`). No sysId or user string interpolated into queries. Zero injection surface. |
| T-06-03 | Information Disclosure | Client fetch (AnalyticsView) | mitigate | Plan 02 adds `X-UserToken: g_ck` header on every analytics fetch (T-06-02 client-side enforcement, mirroring HistoryView.tsx lines 178–183 pattern) |
| T-06-04 | Denial of Service | GlideRecord full-table scan (no limit on bean/method scans) | accept | Home-brew dataset is bounded to hundreds of records maximum. Scans without setLimit are acceptable at this volume. If dataset grows, adding a limit or switching to GlideAggregate (once bug is resolved) is a v2 concern. |
