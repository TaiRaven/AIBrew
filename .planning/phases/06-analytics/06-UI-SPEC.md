## Data Fetch Pattern

Single `useEffect` on mount. Mirrors HistoryView.tsx lines 173–206.

```
GET /api/x_664529_aibrew/analytics/summary
Headers: { Accept: 'application/json', 'X-UserToken': g_ck }
```

Response shape (D-07):
```json
{
  "trend":     [{ "date": "2026-04-30", "rating": 8 }, ...],
  "by_bean":   [{ "name": "Ethiopia Yirgacheffe", "avg": 8.2, "count": 6 }, ...],
  "by_method": [{ "method": "pour_over", "avg": 7.9, "count": 12 }, ...],
}
```

- `trend` is ordered ascending by date, last 30 rated brews (D-05).
- `by_bean` and `by_method` are ordered descending by avg (D-10).
- Unrated brews excluded server-side (D-08).
- State: `{ data: null, loading: true, error: null }` → after fetch resolves/rejects.

---

## Loading State

While the single API call is in-flight, render three skeleton blocks — one per section — using the same skeleton pattern as HistoryView.tsx lines 344–356:

```
height: 120px, background: var(--aibrew-ink-4), borderRadius: 8px, opacity: 0.15, marginBottom: 16px
```

Three skeleton blocks (one per section) with heights 120px / 80px / 80px to approximate the chart areas. No spinner. No text. Matches project skeleton convention.

---

## Error State

Shown only if the fetch itself fails (network error or non-2xx from the API). Replaces the entire view content (not per-section).

```
container: padding var(--sp-md), border: 2px solid var(--aibrew-ink), borderRadius: 6px
text: "Could not load analytics — try again." in var(--aibrew-destructive), fontFamily var(--aibrew-font-body), fontSize 16px
```

Pattern matches HistoryView.tsx lines 362–374.

## Rating Trend Chart (SVG Polyline)

Source: D-02, D-05, D-13.

### Viewport

```
width: 100%
height: 160px
viewBox: "0 0 320 160"
preserveAspectRatio: "none"
```

The viewBox width of 320 corresponds to a 320px phone-width column. `preserveAspectRatio="none"` lets it stretch to fill the section width. The chart occupies the inner region after padding margins.

### Inner chart margins (within the viewBox)

```
top:    12px   (dot overflow room)
right:  16px   (avg score label room)
bottom: 28px   (x-axis date label row)
left:   28px   (y-axis tick number column)
```

Inner chart area: x=28..304, y=12..132 (width=276, height=120).

### Y-axis (rating 1–10)

- Data range is always 1–10 (no dynamic scaling needed — the rating field is 1–10 by design).
- Render 5 tick marks at rating values: 2, 4, 6, 8, 10.
- Tick lines: horizontal `<line>` from x=26 to x=304, stroke `var(--aibrew-ink-4)`, strokeWidth 0.5, opacity 0.5.
- Tick labels: `<text>` at x=24, text-anchor="end", fontSize=12, fill=`var(--aibrew-ink-3)`, font-family Caveat.
- Y pixel = `132 - ((rating - 1) / 9) * 120`. Rating 1 → y=132, rating 10 → y=12.

### X-axis (date labels)

- Show at most 5 date labels evenly distributed across the data points, regardless of how many brews are in the window (up to 30).
- Indices to label: 0, floor(n/4), floor(n/2), floor(3n/4), n-1 (first, quarter, mid, three-quarter, last).
- Format: "MMM D" (e.g. "Apr 30") using `new Date(date).toLocaleString('en-US', {month:'short', day:'numeric'})`.
- Labels: `<text>` at y=152, text-anchor="middle", fontSize=12, fill=`var(--aibrew-ink-3)`, font-family Caveat.
- If n < 5, label only the available data points.

### Polyline

- `<polyline>` connecting all data points.
- stroke: `var(--aibrew-accent)` (#c2410c).
- strokeWidth: 2.
- fill: none.
- Points: calculated from date index (x) and rating value (y) using inner chart dimensions.
- x pixel = `28 + (i / (n - 1)) * 276` where i is the 0-based index and n is the count.

### Dots

- `<circle>` at each data point.
- r: 4.
- fill: `var(--aibrew-accent)` (#c2410c).
- stroke: `var(--aibrew-paper)` (#fbf8f2) to create a white ring separating overlapping dots.
- strokeWidth: 1.5.

### Placeholder (< 2 rated brews — D-13)

```
container: border: 2px dashed var(--aibrew-ink-4), borderRadius: 8px, padding: var(--sp-xl), textAlign: center
text: "No rated brews yet — rate a brew to see your trend."
style: fontSize 16px, color var(--aibrew-ink-3), fontFamily var(--aibrew-font-body)
```

## Horizontal Bar Chart (Avg by Bean / Avg by Method)

Source: D-03, D-10, D-11, D-14.

### Layout

Each chart is a vertically stacked list of rows. No SVG wrapper needed — built from `div` rows.

- Section max width: 100% of container.
- Each row: `display: flex; align-items: center; marginBottom: var(--sp-sm)` (8px).

### Row anatomy

```
[Name label 40% width] [Bar track 100%-40%-48px] [Score label 48px right-aligned]
```

Three columns in each row using `display: flex`:

**Name column (left):**
- Width: 40% of row width (`flex: 0 0 40%`).
- Text: bar label (bean name or method display label).
- fontSize: 14px, fontWeight: 600, color: `var(--aibrew-ink)`, fontFamily: Caveat.
- Overflow: `textOverflow: ellipsis; overflow: hidden; whiteSpace: nowrap`.
- Below the name: brew count subtitle `"N brews"`, fontSize: 12px, color: `var(--aibrew-ink-3)`, fontWeight: 400.

**Bar column (center):**
- `flex: 1; margin: 0 var(--sp-sm)` (0 8px).
- Track: `height: 8px; background: var(--aibrew-ink-4); borderRadius: 4px; opacity: 0.25` — full width of column.
- Bar fill: `position: absolute` inner div, `height: 8px; background: var(--aibrew-accent); borderRadius: 4px`.
- Bar width percentage: `(avg / 10) * 100%` — max rating is 10.
- Track container: `position: relative` to contain the absolute fill bar.
- Row height (with name + subtitle): approximately 36px per row.

**Score column (right):**
- Width: 48px (`flex: 0 0 48px`), text-align: right.
- Text: avg formatted to 1 decimal place, e.g. "8.2".
- fontSize: 14px, fontWeight: 600, color: `var(--aibrew-ink)`, fontFamily: Caveat.

### Method display labels

Map method ChoiceColumn keys to display labels (same list as HistoryView.tsx lines 13–21):

## Interaction Contract

This view is entirely read-only. No user inputs, no modals, no mutations.

| Interaction | Behaviour |
|-------------|-----------|
| Tab navigation to Analytics | Fetch fires once on mount; skeleton shown during load |
| No data | Each section renders its own placeholder independently (D-11) |
| Fetch error | Full-page error message replaces all three sections |
| Tap on chart | No action — charts are not interactive (v1 scope) |
| Scroll | Single scrollable column (D-04); browser native scroll |
