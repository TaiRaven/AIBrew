Read `src/client/app.tsx` and verify the `renderContent()` function imports and renders `HomeView` and `CatalogView` (not placeholders). If placeholders remain from earlier plans, replace them now.

**Step 2 — Final build check.**

```bash
npx @servicenow/sdk build
```

Must exit 0 with no TypeScript errors. If there are errors, fix them before proceeding to install.

**Step 3 — Deploy to the instance.**

```bash
npx @servicenow/sdk install
```

This pushes all Fluent artifacts (tables, role, ACLs, navigator module, UiPage) and the client bundle to `dev203275.service-now.com`.

Watch for any install errors. Common errors to check:
- Scope mismatch: table names in code don't match instance scope — verify `now.config.json` scope matches what was retrieved in PLAN-01
- Endpoint conflict: UiPage endpoint already exists on instance — if so, the SDK will overwrite it (expected behaviour)
- Role conflict: if `x_<scope>_aibrew.user` role already exists, SDK updates it (expected)

**Step 4 — Verify the navigator module is visible.**

Open a browser and navigate to `dev203275.service-now.com`. Log in as an admin user. Check the application navigator for an "AIBrew" menu entry. Click it. The React app should load at `x_<scope>_aibrew_home.do`.

If the navigator entry does not appear:
- Check: is `active: true` set in `aibrew-menu.now.ts` navigator Record data block?
- Check: does the `sys_app_module` record on the instance have `active = true`?
- Check: does the logged-in user have the `x_<scope>_aibrew.user` role?
