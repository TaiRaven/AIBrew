## Threat Model

| Boundary | Description |
|----------|-------------|
| Browser → Table API | All reads/writes pass through the ACL layer; role membership gates access |
| Navigator → UiPage | Page URL is accessible to any logged-in user; data is protected by table ACLs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Elevation of Privilege | x_<scope>_roaster table, x_<scope>_equipment table | mitigate | Acl() declarations with `roles: [aibrew_user]` on read/write/create/delete for each table — see T01 below |
| T-01-02 | Elevation of Privilege | Admin bypass during development | mitigate | PLAT-02 gate: non-admin ACL test required before phase sign-off (01-PLAN-06) |
| T-01-03 | Tampering | SDK deploy overwriting instance edits | accept | Single developer; all fixes go through local source + sdk install. Never edit on instance. |
| T-01-04 | Tampering | Hard-delete of reference records | mitigate | Archive-only (active=false) enforced in UI; no DELETE endpoint exposed; soft-delete pattern in schema |
| T-01-05 | Information Disclosure | Table API accessible without role | mitigate | Four Acl() records per table (read, write, create, delete) block unauthenticated table access |

**Step 3 — Create the navigator module.**

Create `src/fluent/navigator/aibrew-menu.now.ts`:

```typescript
import '@servicenow/sdk/global'
import { ApplicationMenu, Record } from '@servicenow/sdk/core'
import { aibrew_user } from '../roles/aibrew-user.now'

export const aibrew_menu = ApplicationMenu({
  $id: Now.ID['aibrew_menu'],
  title: 'AIBrew',
  description: 'Home coffee brew logging',
  hint: 'Log and track your brews',
  roles: [aibrew_user],
  active: true,
})

export const aibrew_home_module = Record({
  $id: Now.ID['aibrew_home_module'],
  table: 'sys_app_module',
  data: {
    title: 'AIBrew',
    application: aibrew_menu.$id,
    link_type: 'DIRECT',
    query: 'x_snc_aibrew_home.do',    // MUST match UiPage endpoint below (use real scope prefix)
    hint: 'Open AIBrew',
    roles: 'x_snc_aibrew.user',        // comma-separated string for sys_app_module (quirk: not an array)
    active: true,                       // CRITICAL: must be explicit — default is false on sys_app_module
    order: 100,
  },
})
```

Critical rules (from Pitfall 2 and 3):
- `active: true` MUST be in the data block — sys_app_module defaults to inactive.
- `link_type: 'DIRECT'` is required for UiPage navigation.
- `query` must exactly match the `endpoint` value in the UiPage definition (Step 4).
- `roles` on `sys_app_module` is a plain string, not an array — platform quirk.
