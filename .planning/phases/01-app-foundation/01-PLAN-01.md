This task requires human execution because `sdk init` is an interactive command that scaffolds the project and cannot be reversed cleanly.

**Step 1 — Retrieve the real company code from the instance.**

The scope prefix is derived from `glide.appcreator.company.code`. Retrieve it using the authenticated REST API (instance is dev203275.service-now.com, authenticated via OAuth):

```bash
# Run from AIBrew project root (C:\Users\William.Wood\AIBrew)
curl -H "Authorization: Bearer $(npx now-sdk auth --token dev203275.service-now.com 2>/dev/null)" \
  "https://dev203275.service-now.com/api/now/table/sys_properties?sysparm_query=name%3Dglide.appcreator.company.code&sysparm_fields=value" \
  | jq -r '.result[0].value'
```

If the curl/auth approach fails, navigate to the instance UI:
- Login to dev203275.service-now.com as admin
- Navigate to: System Properties > All Properties (filter by name = `glide.appcreator.company.code`)
- Note the value (e.g., `snc`, `will`, `devXXX`, etc.)

**Record the company code** — this becomes the middle segment of the scope prefix: `x_<company_code>_aibrew`

Example: if company code is `snc`, the scope name is `x_snc_aibrew` and the prefix for all table/artifact names is `x_snc_aibrew`.

**Step 2 — Run sdk init from the project root.**

```bash
# Replace <REAL_COMPANY_CODE> with the value from Step 1
cd C:/Users/William.Wood/AIBrew
npx @servicenow/sdk init \
  --appName "AIBrew" \
  --packageName "aibrew" \
  --scopeName "x_<REAL_COMPANY_CODE>_aibrew" \
  --template base
```

Wait for the command to complete. It will create:
- `now.config.json` — scope, scopeId, app name
- `package.json` — with @servicenow/sdk, react, react-dom, @servicenow/react-components
- `tsconfig.json` (root) and nested tsconfigs
- `src/fluent/index.now.ts` — Fluent entry point
- `src/fluent/example.now.ts` — example artifact (DELETE this after init)
- `src/server/script.ts` — sample server script (DELETE this — not used in Phase 1)
- `src/client/` directory may or may not be scaffolded depending on template

**Step 3 — Install npm dependencies.**

```bash
npm install
```

**Step 4 — Delete the sample files that sdk init generates (they cause build conflicts).**

```bash
rm src/fluent/example.now.ts
rm src/server/script.ts
```

If the `src/client/` directory was NOT scaffolded by init, create it:
```bash
mkdir -p src/client/utils src/client/components
```

**Step 5 — Verify the build compiles (empty project).**

```bash
npx @servicenow/sdk build
```

The build should succeed with no errors (possibly zero output files, which is fine at this stage).

**Step 6 — Record the real scope prefix.**

Open `now.config.json` and note the `scope` field value (e.g., `x_snc_aibrew`). Write it down — every subsequent task in Phase 1 will use this exact string. Never use the placeholder `x_SCOPE`.
