# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: App Foundation.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
> Platform: ServiceNow Polaris (Fluent/now-sdk), React 18.2.0.
> Component library: @servicenow/react-components — NO shadcn, NO Tailwind, NO raw HTML elements.

## 1. Design System

| Property | Value | Source |

## 2. Color Contract

60 / 30 / 10 split. All values are app-layer tokens layered over ServiceNow's `--now-*` platform tokens.

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `--aibrew-paper` | `#fbf8f2` | Page background, form backgrounds, modal backdrops |
| Secondary (30%) | `--aibrew-paper-2` | `#f2ede4` | Cards, nav bar surface, catalog section header, table row hover |
| Accent (10%) | `--aibrew-accent` | `#c2410c` | See reserved list below — never used as a background fill |
| Destructive | `--aibrew-destructive` | `#b91c1c` | Archive confirmation button only |

**Accent reserved for (exhaustive list — do not add new uses without updating this spec):**
- "New Roaster" and "New Equipment" primary action buttons (filled)
- Active tab indicator underline
- Chip borders on active/selected state
- Focus ring on inputs (form fields in create/edit views)

**Text colors:**
- Primary text: `--aibrew-ink` (#1d1a17) — headings, labels, card names
- Secondary text: `--aibrew-ink-3` (#6b5c4e) — location, metadata, captions
- Disabled/placeholder text: `--aibrew-ink-4` (#9e8874)
- Link text: `--aibrew-accent` (#c2410c) on `--aibrew-paper` backgrounds — contrast ≥ 4.5:1 required (verify)

**Borders:**
- Card border: 1px solid `--aibrew-ink` (#1d1a17) at 20% opacity — `rgba(29,26,23,0.2)`
- New-item dashed card: 2px dashed `--aibrew-ink-3` (#6b5c4e) at 50% opacity
- Dividers: 1px dashed `--aibrew-ink-4` at 40% opacity

## 3. Spacing Scale

Uses multiples of 4px. Applied via inline style or CSS custom properties — not Tailwind classes.

| Token | Value | Usage |
|-------|-------|-------|
| `--sp-xs` | 4px | Icon-to-label gap, chip internal padding (horizontal) |
| `--sp-sm` | 8px | Chip vertical padding, avatar-to-name gap, list row gap |
| `--sp-md` | 16px | Card internal padding, form field spacing, tab label padding |
| `--sp-lg` | 24px | Section heading margin-bottom, card grid gap |
| `--sp-xl` | 32px | Page-level top padding, modal internal padding |
| `--sp-2xl` | 48px | Between major sections on home screen |
| `--sp-3xl` | 64px | Not used in Phase 1 |

**Exceptions:**
- Touch targets (buttons, tab items, action chips): minimum 44px height — overrides default component sizing if needed.
- Nav tab bar total height: 48px.
- Avatar/initials circle: 40px diameter.
- Card border-radius: 8px.
- Chip border-radius: 16px (fully rounded pill).
- Modal max-width: 480px on desktop; 100vw − 32px on mobile.

## 4. Typography

Four roles. All use `--aibrew-font-disp` (Caveat) for Display and Heading; `--aibrew-font-body` for Body and Label; `--aibrew-font-mono` for numeric values.

| Role | Font | Size | Weight | Line Height | Usage |
|------|------|------|--------|-------------|-------|
| Display | Caveat | 28px | 600 (semibold) | 1.2 | Page titles (Roasters, Equipment), modal headings |
| Heading | Caveat | 20px | 600 (semibold) | 1.2 | Card roaster names, section sub-headings |
| Body | Caveat | 16px | 400 (regular) | 1.5 | Card metadata, notes, form field values, empty-state copy |
| Label | Caveat | 14px | 400 (regular) | 1.4 | Chips, form field labels, secondary metadata, disabled tile labels |
| Mono | Courier New | 14px | 400 | 1.4 | Not used in Phase 1; reserved for brew numeric values (Phase 4) |

**Application rules:**
- Never use more than these 4 size values. Do not introduce 12px, 18px, 24px, or 32px.
- Bold (700) weight is not used. Two weights only: 400 and 600.
- Caveat renders well at all declared sizes on mobile. If Caveat fails to load, fall back to `Georgia, serif` — do not fall back to `sans-serif` (ruins the logbook aesthetic).

## 8. Interaction Contracts

### 8.1 SPA Routing (D-05, locked)

| View | URLSearchParams | Back button |
|------|-----------------|-------------|
| Home | `?view=home` | — |
| Catalog (Roasters) | `?view=catalog&section=roasters` | Goes to `?view=home` |
| Catalog (Equipment) | `?view=catalog&section=equipment` | Goes to `?view=catalog&section=roasters` or `?view=home` |
| Roaster detail | `?view=catalog&section=roasters&id=<sysId>` | Goes to `?view=catalog&section=roasters` |
| Roaster create | Opens modal overlay — no URL change | Modal close returns to list |
| Equipment detail | `?view=catalog&section=equipment&id=<sysId>` | Goes to `?view=catalog&section=equipment` |
| Equipment create | Opens modal overlay — no URL change | Modal close returns to list |

**Polaris iframe detection (D-06, locked):** Every `navigateToView()` call checks `window.self !== window.top` and fires `magellanNavigator.permalink.set` if inside Polaris. See RESEARCH.md Pattern 7.

**popstate handling:** Every view component listens to `window.addEventListener('popstate', ...)` and re-reads `getViewParams()` to re-render. No external router library.

### 8.2 Tab Navigation

| Scenario | Behavior |
|----------|----------|
| Click active tab (not current) | Navigates: sets URLSearchParams, re-renders content area |
| Click current active tab | No action (already on this tab) |
| Click disabled tab | No action — event swallowed in handler: `if (!tab.active) return` |
| Direct URL with `?view=brew` | Renders disabled state message: "Brew logging is coming in a future update." — not a blank page |

### 8.3 Archive Flow

1. User clicks "Archive" action chip on a card or the Archive button on the detail view.
2. `NowModal` opens with confirmation copy (see Copywriting Contract §6.4).
3. User clicks "Archive" (red fill button): PATCH request to Table API sets `active=false`.
4. On success: modal closes, list view refreshes (record disappears from active list).
5. On failure: modal closes, inline error message shown: "Couldn't archive — try again in a moment."
6. User clicks "Keep it" (cancel): modal closes, no change.

**No toast/snackbar for archive success** — disappearing from the list is the confirmation.

### 8.4 Form Save / Cancel

| Action | Behavior |
|--------|----------|
| Save (new record) | POST via RecordProvider; on success close modal and refresh list |
| Save (edit record) | PATCH via RecordProvider; on success navigate back to list |
| Cancel (new, in modal) | Close modal, no change, list unchanged |
| Cancel (edit, on page) | Navigate back to list; prompt "You have unsaved changes" via NowModal if `isDirty` |

### 8.5 Catalog Sub-Navigation

| Action | Behavior |
|--------|----------|
| Click "Roasters" sub-tab | Set `?section=roasters`, render Roasters content |
| Click "Equipment" sub-tab | Set `?section=equipment`, render Equipment content |
| Click disabled sub-tab (Beans, Recipes) | No action — same disabled contract as top nav tabs |

### 8.6 Default View on Load

- No URL params → render `?view=home` (Home Screen)
- `?view=catalog` with no `&section` → default to `?view=catalog&section=roasters`
- Unknown `?view` value → redirect to `?view=home`

## 9. Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Minimum touch target | 44px height for all interactive elements (tabs, chips, buttons, modal close) |
| Color contrast (text on background) | `--aibrew-ink` (#1d1a17) on `--aibrew-paper` (#fbf8f2) = ~14.5:1 (AAA) |
| Color contrast (secondary text) | `--aibrew-ink-3` (#6b5c4e) on `--aibrew-paper` (#fbf8f2) — verify ≥ 4.5:1 before ship |
| Color contrast (accent on paper) | `--aibrew-accent` (#c2410c) on `--aibrew-paper` (#fbf8f2) — verify ≥ 4.5:1 (use WebAIM) |
| ARIA roles — tabs | `role="tablist"` on nav container, `role="tab"` per tab, `aria-selected`, `aria-disabled` on disabled tabs |
| ARIA roles — modal | `role="dialog"`, `aria-modal="true"`, `aria-labelledby` pointing to modal heading |
| ARIA roles — cards | `role="article"` or default div; action buttons have descriptive `aria-label` ("Edit Acme Coffee", not just "Edit") |
| Keyboard navigation | Tab key moves through chips/buttons in card; Enter activates; Escape closes modal |
| Focus management | On modal open: focus moves to first interactive element (Name field). On modal close: focus returns to triggering element (New card). |
| Disabled tabs | `aria-disabled="true"` — screen reader announces "Brew, dimmed" without requiring visual-only cues |
| Reduced motion | No CSS animations in Phase 1 — no `prefers-reduced-motion` handling required |

## 10. Platform Notes

### 10.1 Polaris iframe detection (D-06 — non-negotiable)

Every page (every React component that triggers navigation) must call `navigateToView()` from `src/client/utils/navigate.ts`. This utility performs `window.self !== window.top` detection and fires the Polaris permalink event. Do not navigate with `window.location.href =` or `<a href>`.

### 10.2 Array.from polyfill (D-06 — non-negotiable)

The polyfill is inline in `index.html`, `type="text/javascript"`, before the `type="module"` script. No CDATA wrappers per D-06. See RESEARCH.md Pattern 6 for exact code. If blank-page issues arise, CDATA wrappers are the first diagnostic step.

### 10.3 No raw HTML elements

`<button>`, `<input>`, `<select>`, `<a>` are forbidden. Use `NowButton`, `NowTextInput`, `NowSelect`, and navigation via `navigateToView()` only.

### 10.4 CSS tokens vs. Tailwind

Do not use Tailwind. Apply styles via CSS custom properties (`:root` tokens declared as shown in §1) and inline `style` props on React components, or a plain `.css` file imported in `main.tsx`. ServiceNow's `--now-*` tokens are used only where the platform component exposes them — custom app tokens use `--aibrew-*` prefix to avoid collisions.

### 10.5 window.confirm prohibition

`window.confirm()` and `window.alert()` are blocked in the Polaris iframe context. All confirmations (archive, unsaved changes) must use `NowModal`.
