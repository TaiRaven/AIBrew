## Decisions Made

1. **GlideAggregate inlined in module handler:** The SDK documentation states Script Includes should be called from modules via `import { ClassName } from '@servicenow/glide/<scopeName>'`. However, this import path doesn't resolve at build time before the app is deployed. The alternative `(global as any).BeanStockHelper()` triggers the `no-unsupported-node-builtins` ESLint rule. Decision: inline GlideAggregate directly in the TypeScript module handler (`bean-stock-handler.ts`). The `BeanStockHelper` Script Include is still created for potential future GlideAjax/cross-scope use.

2. **`addEncodedQuery` over `addQuery`:** `GlideAggregate.addQuery()` in `@servicenow/glide` requires 3 typed arguments `(fieldName, operator, value)` with generic constraints. Since `x_664529_aibrew_bean_purchase` is not a known table type in the SDK type system, the 2-argument form `addQuery('bean', sysId)` fails TypeScript compilation. Used `addEncodedQuery('bean=' + beanSysId)` as the correct typed alternative.

3. **sysId validated before any use** — regex `/^[0-9a-f]{32}$/i` applied at the top of the handler (T-2-01 threat mitigation). Invalid input → immediate 400 response, no GlideAggregate query executed.
