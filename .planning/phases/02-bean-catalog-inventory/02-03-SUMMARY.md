## Decisions Made

1. **BeanDetailView stub includes archive modal pattern:** Acceptance criteria required `e.detail?.payload?.action?.label` to be present in the file. 2. **CatalogView beans tab enabled in this plan:** The beans sub-tab was `disabled: true` in CatalogView.tsx. 3. **StockBar as internal sub-component:** The stock progress bar logic is self-contained enough to warrant its own function (`StockBar`) for clarity and reuse within the file, but not exported — no cross-component sharing needed in Phase 2.

### Auto-fixed Issues

**1. [Rule 1 - Bug] TS6133 "declared but never read" on imported table constants in bean.now.ts and bean-purchase.now.ts**
- **Found during:** Task 1 — first SDK build attempt
- **Issue:** `bean.now.ts` imports `x_664529_aibrew_roaster` and uses only the string `'x_664529_aibrew_roaster'` in `ReferenceColumn({ referenceTable: ... })`. TypeScript's `noUnusedLocals` fires because the import is not referenced as a JS value. Same issue in `bean-purchase.now.ts` for `x_664529_aibrew_bean`.
- **Fix:** Changed `referenceTable: 'x_664529_aibrew_roaster'` → `referenceTable: x_664529_aibrew_roaster.name` (and equivalent for bean-purchase). The `.name` property is the string table name set in the `Table({ name: ... })` call — semantically identical, but satisfies TypeScript's unused-locals check.
- **Files modified:** `src/fluent/tables/bean.now.ts`, `src/fluent/tables/bean-purchase.now.ts`
- **Commit:** 4d3ab3a
